Asynchronous Process Call Injection (Windows)

Executing shellcode within a process.

This entry aims to explain what is an Asynchronous Process Call, how does it work and how it can be exploited to execute shellcode. I. Definition of an APC +

An Asynchronous Procedure Call (APC) is a function that executes asynchronously in the context of a particular thread. On simpler terms, it's a queued code stored within a specific part of a process that runs when such part is free of any work. On even simpler terms, it's a method used by computers that allows execution without having to wait for a task to finish.

II. Lifecycle of an APC +

To fully grasp the inner workings of APC, it's key to understand it's flow, which can be divided in the following steps:

1. Invocation
The main program invokes a function which is handled to a system API.
2. Exchange
The API captures the parameters and returns control to the program which received a placeholder (handle).
3. Background Execution
The API takes care of the task in the background (async), allowing the program to continue executing lines of codes (sync).
4. Completion
Once the task is completed, the API returns a "notification" into a queue.
5. Verification
As the program completes other tasks (sync), the queue transfers the result back to it, replacing the placeholder.


III. What is an APC Injection? +

An APC Injection is a type of code injection which involves running a payload within the address space of a legitimate process, a thread to be exact. There are many variants of this type of injection: AtomBombing, Early Bird, NtQueueApcThread...
